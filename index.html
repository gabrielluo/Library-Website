So what is a version control system?

If you have used Microsoft Word before, you must have this experience:

You wanted to delete a paragraph, but being afaird of losing it permanently. So you used "save as ......" and created a new Word document, and then went on changing more, you need to "save as ......" another file... Ultimately your folder became this:

A week later, you wanted to retrieve a block of deleted text, but could not remember which files was the desired one, then you needed to take a deep search into every files, which is really troublesome.

To make matter worse, when you needed help from your colleague, you copied the file to a USB driver to her (or email or something), and continued to modify that .doc files. One day later, your colleague send back her file to you, at this time, you have to think about what changes have her made and what is your parts that ought to be preserved.

If there is a software that can help you to record changes automatically each time anyone edit the file, you can let colleagues edit a same file collaboratively, without messing it up when passing it around. If you want to see a particular change you can do it with a simple glance, isn't that super convenient? This record turns out to be:

Version    User    Description                          Date
1          Jack    removes softwareTerms of Service     7/12 10:38
2          Bob     adds the limit of License            7/12 18:09
3          Jack    adjusted the contract amount         7/13 9:51
4          Jack    extended the free upgrade cycle      7/14 15:17

By this time, you quit manage multiple "versions" manually and started a brave world of version control.
